# 🔀 Route-Split | 网络分流工具

> 一款专为 Windows 内外网卡环境设计的网络分流工具，解决企业内网与外网同时使用时的路由冲突问题。

---

## 📋 项目背景

在企业办公环境中，经常需要同时连接：
- **外网**：通过公司网络或代理访问互联网
- **内网**：访问公司内部系统（如 OA、ERP、内部网站）
- **VPN/代理**：如 FlClash、Clash 等科学上网工具

当多张网卡同时工作时，Windows 的默认路由策略可能导致：
- 内网地址走了外网出口，无法访问
- VPN 抢占了所有流量，内网不通
- 系统代理干扰了正常的网络分流

**Route-Split** 通过智能路由配置，一键解决这些问题。

---

## ✨ 功能特性

### 🔌 网卡智能识别
- 自动探测系统所有 IPv4 网卡
- 显示网卡名称、本地 IP、网关信息
- 支持手动指定内网/外网出口

### 🚀 一键配置
- 自动添加内网路由规则（如 `10.0.0.0/8` → 内网网关）
- 自动关闭系统代理，避免干扰
- 自动刷新 DNS 缓存

### 📊 实时状态监控
- 显示当前内网/外网路由状态
- 显示系统代理开启状态
- 路由规则生效状态实时反馈

### 📖 FlClash 配置指南
- 内置 `fake-ip-filter` 配置模板
- 一键复制到剪贴板
- 动态生成（根据自定义网段）

### 🎨 现代化界面
- 深色/浅色主题自动切换
- 响应式布局，支持窗口缩放
- 中文原生支持

## 📦 安装与使用

### 方式一：下载预编译版本

从 [Releases](../../releases) 页面下载最新的 `route-split.exe`，右键以管理员身份运行。

### 方式二：从源码编译

```bash
# 克隆仓库
git clone https://github.com/GIWTO/route-split.git
cd route-split

# 编译 Release 版本
cargo build --release

# 运行（需要管理员权限）
./target/release/route-split.exe
```

### 方式三：本地开发/运行

```bash
# 直接运行（建议在管理员权限的终端下执行）
cargo run
```

## 🛠️ 使用指南

### 基本流程

1. **以管理员身份运行**程序
2. 在「网卡流向手动配置」中选择：
   - 🌐 外网出口卡：用于访问互联网的网卡
   - 🏢 内网分流卡：用于访问公司内网的网卡
3. 确认「自定义内网目标网段」（默认 `10.0.0.0/255.0.0.0`）
4. 点击「🚀 一键配置」

### 高级配置

**自定义内网网段**：
- 如果公司内网使用 `192.168.x.x`，可修改为 `192.168.0.0` / `255.255.0.0`
- 如果公司内网使用 `172.16.x.x`，可修改为 `172.16.0.0` / `255.240.0.0`

**FlClash 配置**：
将「排除指南」中的配置复制到 FlClash 的 `fake-ip-filter` 配置中，确保内网域名不被代理。

## ⚠️ 注意事项

1. **必须以管理员身份运行**，否则无法修改路由表
2. 路由规则使用 `-p` 参数，**重启后仍然生效**
3. 如需恢复，点击「↩ 回滚配置」即可清除添加的路由规则
4. 本工具不会修改网卡的 IP 配置，仅操作路由表和系统代理设置
